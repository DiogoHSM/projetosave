# UX_RULES.md

## Objetivo
Regras de UX para manter consistência e reduzir erros do usuário.

## Regras gerais
- Mostrar contexto ativo (org + modo) em local visível.
- Confirmar ações irreversíveis (encerrar discipulado, revogar convite, remover membro).
- Feedback claro de estados (draft, submitted, in_review, needs_changes, approved).
- Sempre explicar por que algo está bloqueado (ex: "assinatura inativa").
- Não expor termos técnicos desnecessários.

## Padrões de telas
- Listas com busca e filtros (status, grupo, período).
- Estados vazios com instrução do próximo passo.
- Loading com skeleton (onde fizer sentido).
- Erros com mensagem acionável.

## Acessibilidade e linguagem
- Linguagem simples e respeitosa.
- Evitar jargões de TI.
- Suporte a mobile (responsivo).
